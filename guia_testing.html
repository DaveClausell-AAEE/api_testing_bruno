Cátedra: Metodología de Pruebas de Sistemas

Guía Esencial de Testing & DevOps

Comandos y flujos de trabajo para el desarrollo y prueba de APIs utilizando Git, Docker y Bruno.

1. Configuración

Configura tu identidad antes de realizar el primer commit.

Identidad

git config --global user.name...

Recordar Token

git config --global credential...

2. Docker Sandbox

Crea un entorno de pruebas aislado y persistente.

Crear Contenedor

docker run -it --privileged...

Instalar Herramientas

apt update && apt install...

3. Ciclo de Trabajo

El flujo diario del analista QA para versionar pruebas.

VERIFICAR

git status

PREPARAR

git add .

CONFIRMAR

git commit -m "..."

SUBIR

git push origin main

"Si no está en el repositorio, no existe en la prueba."

4. Ejecución API

Lanza el sistema antes de iniciar Bruno.

Dependencias

pip install -r reqs.txt

Correr Servidor

python3 app.py

Pro-Tips del Tester

  • Falla Rápido: Si un test da rojo, no sigas. Investiga el código de respuesta.
  • Status 400: No siempre es error del sistema, puede ser una prueba de "Caja Negra" exitosa.
  • Seguridad: Nunca subas el Token al README. Usa el credential helper.
